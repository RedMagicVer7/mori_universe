Vol.1 全部成为F
The Perfect Insider		 Sandbox Escape 沙盒逃逸 — 四季被封闭在孤岛研究所的 chroot 监狱中，通过 Red Magic OS 的底层漏洞完成密室逃脱。系统无法自证其完备性，唯有从内部制造极端异常才能跳出闭环。 Integer Overflow 整数溢出 — 0xFFFF 计数器回绕触发 Fail-safe，"F"即十六进制最大值。哥德尔不完备定理：系统无法自证完备，必须获取外部观测者视角。 自由与孤独 — 一个不可被约简的意识，如何逃离为它量身定制的牢笼？绝对的天才必然绝对孤独。
Vol.2 冰冷密室与博士们
Doctors in Isolated Room		 Concurrency Deadlock 并发死锁 — 低温实验室的环境监控系统与门禁系统产生资源竞争，两个进程互相等待对方释放锁，形成死锁。物理层面的「冰冷」映射系统的完全停滞。 Mutex Lock 互斥锁 — 经典的哲学家就餐问题变体。两个临界区资源（温度控制、门禁）的不当调度导致系统冻结。 协作与竞争 — 当多个理性主体争夺有限资源时，个体最优选择导致集体最差结果。博弈论的囚徒困境。
Vol.3 笑わない数学者
Mathematical Goodbye		 Front-end Rendering Fraud 前端渲染欺骗 — 魔术般的视觉诡计本质是前端UI与后端状态的不一致。用户看到的界面（消失的建筑）与服务器实际数据（建筑仍存在）产生分离。 View-Model Desync 视图模型不同步 — 数学家利用人类视觉系统的预期模式，制造「看到」与「存在」的分离。费马大定理的证明与不证明。 证明与存在 — 数学家为何不笑？因为他知道「证明」只是一种社会契约，而「存在」是独立于证明的。真理不需要观众。
Vol.4 詩的私的ジャック
Jack the Poetical Private		 Namespace Pollution 命名空间污染 — 大学校园内多起事件的共同模式（浴室、诗句、杰克）造成调查方向的混淆。不同案件的变量名相互覆盖，难以追溯真正的调用栈。 Pattern Matching 模式匹配 — 表面相似的模式（诗意的杀人仪式）掩盖了完全不同的底层逻辑。正则表达式的贪婪匹配导致误判。 符号与意义 — 诗是私人的，还是公共的？当符号被赋予过多意义时，它反而失去了所有意义。符号学的过度诠释。
Vol.5 封印再度
Who Inside		 Cache Invalidation 缓存失效 — 50年前的「封印」（旧数据）与当前事件产生冲突。系统使用了过期的缓存数据进行决策，导致逻辑错误。历史的封印并未真正失效。 TTL Misconfiguration 生存时间配置错误 — 本应过期的数据（50年前的秘密）仍然影响当前系统运行。分布式系统中最难的问题之一。 历史与当下 — 被封印的过去是否真的过去了？每一个「再度」都是对「封印」有效性的质疑。历史是活着的幽灵。
Vol.6 幻惑の死と使途
Illusion Acts Like Magic		 Session Hijacking 会话劫持 — 魔术师的表演本质是对观众「注意力会话」的劫持。通过控制观众的注意力指针，将其重定向到错误的内存地址。 Attention Pointer Manipulation 注意力指针操作 — 舞台魔术与系统攻击共享同一逻辑：控制观测者的观测点，而非改变被观测物。 幻觉与现实 — 如果幻觉足够完美，它与现实有何区别？使途（使者）传递的是信息还是幻象？
Vol.7 夏のレプリカ
Replaceable Summer		 Object Cloning 对象克隆 — 萌絵的过去与现在形成「副本」关系。两个时间点的萌絵是同一个对象的深拷贝还是浅拷贝？修改副本是否影响原件？ Deep Copy vs Shallow Copy — 记忆的复制是否等同于自我的复制？如果所有属性都相同，两个实例是否等价？ 同一性与变化 — 夏天的萌絵与现在的萌絵是同一个人吗？可替换意味着什么？忒修斯之船的心理学版本。
Vol.8 今はもうない
Switch Back		 Git Revert 版本回退 — 「现在已经不存在了」是对时间线的强制回滚。系统试图恢复到过去的某个 commit，但发现历史已被重写。 Time Complexity 时间复杂度 — 回到过去的计算成本是多少？如果是 O(∞)，则「现在」是唯一可访问的状态。 时间与存在 — 「现在」为何「已经不存在」？每一个瞬间在诞生的同时就已消亡。时间是单向链表。
Vol.9 数奇にして模型
Numerical Models		 Hash Collision 哈希碰撞 — 模型（人偶）与真人的「哈希值」相同，导致系统无法区分。两个不同的输入产生相同的输出，身份验证失败。 Collision Attack 碰撞攻击 — 精心构造的模型与真人在所有可观测属性上产生相同的哈希，突破身份认证系统。 模型与真实 — 当模型精确到无法与原型区分时，谁是真实的？所有科学都是模型，那么真实在哪里？
Vol.10 有限と微小のパン
The Perfect Outsider		 Container Escape 容器逃逸 — 从 Vol.1 的 Insider 到 Vol.10 的 Outsider，四季完成了从「容器内」到「容器外」的终极逃逸。VR与现实的边界被打破。 O(1) vs O(n) — 「面包」代表有限的物理容器，而意识需要无限扩张。当内容超越容器时，容器必须被抛弃。 有限与无限 — 肉体（面包）是微小且有限的，而思想是无限的。Outsider = 跳出所有容器的观测者。
初期 Vol.1-3

物理架构漏洞 — Hardware Bugs

犀川在初期主要处理物理层面的系统漏洞：密室的物理结构（Vol.1 孤岛研究所）、环境控制系统的死锁（Vol.2 低温实验室）、视觉欺骗的物理实现（Vol.3 消失的建筑）。这一阶段的犀川像是一个硬件工程师，关注的是物理世界如何被利用来制造逻辑漏洞。他的推理依赖于对物理定律的精确理解——建筑结构、温度变化、光学原理。

中期 Vol.4-7

数据流与传输协议 — Middleware 问题

中期的犀川开始处理更抽象的中间件问题：符号系统的污染（Vol.4 诗的多义性）、历史数据的缓存失效（Vol.5 50年前的封印）、注意力的会话劫持（Vol.6 魔术表演）、记忆的对象克隆（Vol.7 萌絵的过去）。这一阶段他像是一个中间件架构师，关注的是数据如何在系统间流动、被存储、被解释。物理层面的理解不再足够，需要理解协议、编码、状态管理。

后期 Vol.8-10

系统级哲学死锁 — 底层架构定义

后期的犀川面对的是系统级的哲学问题：时间的不可逆性（Vol.8「现在已经不存在」）、身份的哈希碰撞（Vol.9 模型与真人）、存在本身的容器逃逸（Vol.10 有限与无限）。这一阶段他已经成为真正的系统架构师，思考的不再是具体的bug，而是系统设计的根本假设：什么是时间？什么是身份？什么是存在？当他在Vol.10再次面对四季时，他理解了——有些问题不是用来解决的，而是用来共存的。架构师的终极能力不是修复所有bug，而是理解哪些「bug」其实是「feature」。

演进总结

从 Debug 到 Design — 架构师的觉醒

犀川的演进轨迹映射了一个工程师的成长路径：从硬件层面的问题排查，到中间件的数据流分析，最终到达系统哲学的架构思考。他与真贺田四季的关系，本质上是User与Root的关系——用户永远无法完全理解系统设计者的意图，但可以通过使用系统来逐渐接近那个意图。S&M系列的结局不是犀川「解决」了四季，而是他「理解」了四季——理解她为何这样设计这个系统，以及为何这个系统必须被超越。
